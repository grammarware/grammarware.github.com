CSS: Cascading Style Sheets

Cascading Style Sheets, or CSS for short, is the language that makes the web as colourful and fun as we are used to. It is present in some form on pretty much every webpage and is essential to each company’s recognisability. However, there is very little research made on CSS compared to other less widespread software languages like Java. On this page, I collect references to all the papers published at international peer-reviewed academic conferences and journals, that make CSS main focus of their investigation.

  30. Amaia Aizpurua, Myriam Arrue, Markel Vigo, Julio Abascal, Exploring Automatic CSS Accessibility Evaluation, ICWE 2009. (bibtex) (open access) (search)
      Extending existing accessibility evaluation procedures to CSS while keeping them fully automatic. The authors list 6 different kinds of smells (called “test cases” by accessibility researchers) and specify them in a guidelines specification language called UGL. Several coding conventions are discussed, and their harmfulness explained in detail.
  31. Miško Hevery, Adam Abrons, Declarative Web-Applications without Server: Demonstration of How a Fully Functional Web-Application can be Built in an Hour with only HTML, CSS & <angular/> JavaScript Library, OOPSLA 2009 Demo. (bibtex) (search)
      A salespitch for what now is called AngularJS; nothing about CSS whatsoever.
  32. Kehinde Alabi, Generation, Documentation and Presentation of Mathematical Equations and Symbolic Scientific Expressions using Pure HTML and CSS, WWW 2007. (bibtex) (open access) (search)
      This two-page paper accompanying a poster, explains how the authors made a tool to display mathematical formulae in pure HTML and CSS, claiming it to be more portable than the alternatives such as MathML. The author uses table layout which in the meantime has been totally outlawed by web designers and developers, and GIFs instead of Unicode characters. The only use for CSS in this project seems to be to make cell borders invisible.
  33. Matt Germonprez, Michel Avital, The Impacts of the Cascading Style Sheet Standard on Mobile Computing, JITSR 4(2), 2006. (bibtex) (open access) (search)
      A highly abstract paper about how CSS and the way it is defined, impacts the future of mobile computing.
  34. Simon Harper, Sean Bechhofer, Darren Lunn, SADIe: Transcoding based on CSS, ASSETS 2006. (bibtex) (open access) (search)
      SADIe is Structural Semantics for Accessibility and Device Independence. This is a very short paper briefly explaining the challenges visually impaired people face when surfing the web. The solution presented here which uses ontological inference and reasoning to refactor existing webpages into simplified user-friendly equivalents.
  35. Alessandro Cogliati, Petri Vuorimaa, Optimized CSS Engine, WEBIST 2006. (bibtex) (search)
      Details of implementing and optimising a browser engine for CSS. The resulting system is 25%–45% faster and uses 85% less memory. [9,7,5] are parts of the same project called X-Smiles (2000–2006).
  36. Henrik Stormer, Personalized Websites for Mobile Devices using Dynamic Cascading Style Sheets, IJWIS 1(2), 2005. (bibtex) (search)
      The paper lists limitations of mobile devices of its time (96 x 65 pixel screens, etc) and proposes to give up on the one-website-fits-all paradigm and generate personalised content from manually created baseline pages. Their case study is a webshop engine. Best quote from the paper: “An interesting development are smartphones”. Amusing read, but hopelessly outdated ideologically and full of misspellings.
  37. Mikko Pohja, Petri Vuorimaa, CSS Layout Engine for Compound Documents, LA-WEB 2005. (bibtex) (search)
      Disregarding Mozilla, Opera, Konqueror and IE as wanna-be XML browsers suffering from HTML legacy, these authors present requirements, design, implementation, integration and evaluation of a stylesheet-based layout engine for a user agent capable of running on anything from handheld devices to digital TV sets.
  38. Henrik Stormer, Personalized Websites for Mobile Devices using Dynamic Cascading Style Sheets, MoMM 2004. (bibtex) (search)
      The Proceedings of MoMM 2004 with ISBN 3-85403-184-X, edited by Gabriele Kotsis, David Taniar and Ismail Khalil Ibrahim, are not available online and I could not procure a copy through other methods. However, this paper is presumably just an earlier version of [8]
  39. Alessandro Cogliati, Mikko Pohja, Petri Vuorimaa, XHTML and CSS Components in an XML Browser, ICOMP 2003. (bibtex) (search)
      When HTML was redefined in XML terms as XHTML, it became challenging again to make a browser and interesting enough to write a paper about it. This paper is introductory and has three follow-ups: “An XHTML 2.0 Implementation”, [7] and [9] above.
  40. Håkon Wium Lie, Janne Saarela, Multipurpose Web Publishing Using HTML, XML, and CSS, CACM 42(10), 1999. (bibtex) (open access) (search)
      The official overview of the CSS1 and CSS2 coauthored by the inventor of CSS. Contains figures of the same content displayed with different styles and speculations about the future of the web.
  41. Greg J. Badros, Alan Borning, Kim Marriott, Peter J. Stuckey, Constraint Cascading Style Sheets for the Web, UIST 1999. (bibtex) (open access) (search)
      This paper is based on the very first, now long-forgotten CSS2.0 proposal which was later replaced by a non-backwards-compatible CSS2.1 which people mean nowadays when they say “CSS2”. The authors propose CCSS, a language that adds constraints to CSS2.0 with which one can express, for instance, that the font size of the headings should be strictly greater than the font size of the paragraphs. They extend the Amaya browser to implement the language and its constraint solving rendering schemes.
  42. Jukka K. Korpela, Lurching Toward Babel: HTML, CSS, and XML, IEEE Computer 31(7), 1998. (bibtex) (open access) (search)
      A broad explanation of what CSS can do, possibly the first published account of the now-classic example of “this is how the web would have looked without styles”.
  43. Stuart Culshaw, Michael Leventhal, Murray Maloney, XML and CSS, WWW Journal 2(4), 1997. (bibtex) (open access) (search)
      One of the very first detailed mentions of CSS less than a year after CSS1 reached W3C Rec level. No research contribution, just explanations of what it can do, with lots of examples. PREPARE FOR CAPSLOCK.
